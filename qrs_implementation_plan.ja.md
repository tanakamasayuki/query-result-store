# QRS 実装方針（PHP / 最小依存）

## 1. 目的

本書は `qrs_requirements_spec.ja.md` を実現するための実装方針（How）を定義する。
要件（What）に矛盾しない範囲で、保守性・移植性・運用容易性を優先する。

------------------------------------------------------------------------

## 2. 基本方針

- UI と Batch を分離する
- PHP はプレーンな構成で実装する
- 外部ライブラリ依存は最小化する
- 古い PHP でも動かせる書き方を優先する
- DB は SQLite / MySQL / PostgreSQL をサポートする

------------------------------------------------------------------------

## 3. 実行アーキテクチャ

### 3.1 UI

- 役割: 設定管理、一覧、手動実行リクエスト登録
- Web 経由でジョブを直接実行しない
- 実行は request を登録するだけに限定する

### 3.2 Batch

- 役割: request の取り込み、Run 作成、Run 実行、リトライ、catch-up
- cron から単一の CLI スクリプトを定期起動する
- スクリプト内の多重起動制御は実装しない
- 多重実行防止は運用側 `flock` 前提とする
- 当面は 1 プロセスで実行し、将来必要なら内部処理を分割可能な構造にする

例:

    * * * * * flock -n /var/lock/qrs-worker.lock php /path/bin/worker.php

------------------------------------------------------------------------

## 4. 推奨ディレクトリ構成

    public/index.php
    bin/worker.php
    lib/Config.php
    lib/Db.php
    lib/RedashClient.php
    lib/SchemaGuard.php
    lib/StorageAdapter/
    lib/Repository/
    lib/Service/

------------------------------------------------------------------------

## 5. PHP 方針

- 互換性重視（目安: PHP 5.3+ で動作可能な記法）
- `PDO` を共通 DB インターフェースとして使用
- 必須拡張は最小限（`pdo_*`, `curl`, `json`）
- Composer なしでも動作可能にする

------------------------------------------------------------------------

## 6. DB 対応方針

- DB差分は Adapter 層に隔離する
- 実装順は SQLite で PoC を先行し、MySQL / PostgreSQL は段階的に追加する
- 差分対象:
  - DDL
  - 識別子クォート
  - メタ情報取得
  - トランザクションの扱い
- 論理テーブル名は長さ制限回避のため短縮規則（ハッシュ併用）を用意する

------------------------------------------------------------------------

## 7. 実行制御方針

- 状態遷移は `queued -> running -> success|failed` を厳密運用
- `snapshot` の DELETE/INSERT は同一トランザクションで処理
- リトライ上限、優先度、catch-up は要件定義どおり実装

------------------------------------------------------------------------

## 8. 非目標

- フレームワーク導入を前提にしない
- 高度なジョブ基盤（メッセージキュー等）を初期導入しない
- 自動スキーママイグレーション機能は実装しない

------------------------------------------------------------------------

## 9. 要件仕様との関係

本書は実装上の方針を定義する補助文書であり、
要件の正本は `qrs_requirements_spec.ja.md` とする。
